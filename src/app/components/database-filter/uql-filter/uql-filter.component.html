Advanced Filter Docs

Write compact filters with readable names.

UQL is a predicate language for filtering inheritances. The editor already supplies the fixed where line, so you write only the condition that should be true for each result.

Start With A Simple Predicate

A predicate is one complete condition. Numeric filters use comparison operators such as >=, =, and <=.

Global Factors Versus Specific Slots

Global factor names search across the whole visible lineage pool, so they can use totals like 6 or 9 stars. Slot filters look at one uma slot, so they use natural 1-3 star values.

White Skills And Array Fields

Skill lists use array operators. Use has for one required skill, has any (...) for at least one listed skill, and has all (...) when every listed skill must exist.

Known skill names are treated as values, even when they contain words such as and, or, in, or all.

Optional And Lineage Sorting

Use optional and lineage white filters when matching skills should influence result order. Add scoring settings as trailing comma attributes.

optional_white(...)
Requires and scores global white skill matches.
optional_main_white(...)
Requires and scores main parent white skill matches.
optional_any_white(...)
Checks global and main parent white skill matches.
lineage_white(...)
Sorts by lineage-style white skill stacking.
type_weight / distinct_weight
Score per distinct requested optional white skill.
level_weight
Score per matched spark level.
match_weight
Score per matched occurrence.
proc_weight
Adds proc-chance weighted scoring for matched skills.
proc_kind
Chance table: 3 skill, 2 race, 4 scenario.
affinity
Overrides row-derived affinity for proc scoring.
stack_weight
Base lineage stack score.
occurrence_weight
Lineage score per occurrence.
base / base_percent
Lineage stack base percent.
decay / decay_percent
Lineage decay percent between requested skills.
weight
Overall multiplier.
